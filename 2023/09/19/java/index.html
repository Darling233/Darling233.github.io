面向对象与面向过程

面向对象（OOP）:

1.物以类聚,先解决问题需要哪些分类,再对分类进行单独思

​ 考.最后对分类下的细节进行面向过程的思考

2.适用于复杂、需要多人协作的问题.

  1. 本质：以类的方式组织代码，以对象的组织（封装）数据。
  2. 三大特性：封装、继承、多态
  3. 对象是具体的事物，类是从对象中抽象出来。类是对象的模板。

方法调用:yin_yang:

  1. 方法：非静态实例化后才存在

  2. 调用：

    ​ a.静态调用非静态：对象类型 对象名 = 对象值（需要先实例化

  3. 值传递与引用传递，值传递是实参传给形参，引用传递传地址？如person.name是一个具体的对象属性，为引用传递，可以改变属性。、

构造器:dagger:

实例化还会有构造器,作用是初始化,若定义了有参构造,无参必须显示定义.通过方法重载选择调用

alt+insert()自动生成构造器

内存简单分析

栈中放方法、和引用一些变量名。堆中放方法区和类。

所有对象都可调用静态方法

小结

类包括属性和方法

类是一个模板，抽象的。而对象是一个具体的实例

对象创建使用：必须使用new关键字创造对象，有构造器；对象的属性操作；对象的方法操作

封装

属性私有,流出get与set供调用

继承

继承是宏观的把握

修饰符:

在java中,都默认继承object类

SUPER

this与super比较:

this:本身调用者这个对象

super:代表父类对象的引用,调用父类构造方法,必须在构造方法的第一个

方法的重写（override）

重写是方法的重写,与属性无关

非静态、继承才有重写

注意：

重写的意义：父类的功能,子类不一定需要

多态

可以实现动态编译

**多态的注意事项:**g

  1. 方法才有多态,属性没有
  2. 类之间要有继承关系、方法需要重写、父类引用指向子类对象
  3. 重写才有多态

不能重写的情况：

  1. static
  2. final，常量
  3. private 方法

instanceof

instanceof：判断是否是某类型，两个类型要为父子关系。

类型转换

低转高类型不需要强制转换

高转低需要（低的类型）待转对象

子类转换为父类，可能丢失一些方法；父转子可能丢失精度

作用：方便方法的调用，通过升级和降级就可以调用一些方法，减少代码

static

static可以多个类调用,适合多线程,因为随类加载

非静态可以直接调用静态方法

abstract关键词

抽象方法,只有方法名字,没有方法的实现

作用:将一些东西抽象出来,再具体实现

接口

interface关键字

异常

finally可以用于io流的关闭

catch中是想要捕获的类型,可以捕获多个异常,从小到大.

自定义异常

继承Exeception类

集合

注解(annotation)

反射

Java反射机制提供的功能

String是引用数据类型,引用数据类型是分配在堆上,基本数据类型是分配在栈上
